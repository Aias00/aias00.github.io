1.线程间同步

为何要使用同步?

java 允许多线程并发控制,当多个线程同时操作一个可共享的资源变量时,将会导致数据的不准确,相互之间产生冲突,因此可以加入同步锁以避免在该线程没有完成对数据的操作之前,数据被其他线程调用,从而保证了该变量的唯一性和准确性

同步的方式:

1.同步方法或同步代码块:synchronized 关键字或 Lock 锁

2.volatile 关键字

3.使用 ThreadLocal

4.阻塞队列

5.使用原子变量

2.java 变量的读写

Java 通过几种原子操作完成工作内存和主内存的交互:

1.lock: 把主内存中的一个变量标志为一个线程独享的状态

2.unlock: 把主内存中的一个变量解除独占状态

3.read: 将主内存中的变量读到工作内存中

4.load: 把 read 操作传过来的变量的值存入工作内存的变量副本中

5.use: 当执行引擎需要使用到一个变量时,将工作内存中的变量的值传递给执行引擎

6.assign: 将执行引擎收的的值赋值给工作内存中的变量

7.store: 将工作内存中的变量的值传到主内存中

8.write: 将 store 得到值放到主内存的变量中

3.JMM 内存屏障

4.volatile 关键字

保持内存可见性:所有线程都能看到共享内存的最新状态

原理:

  • read,load,use 动作必须连续出现.

  • assign,store,write 动作必须连续出现.

  • 1.线程对 volatile 变量进行修改之后,要立即写回主内存,并且会强制让缓存了该变量的线程中的数据失效

  • 2.线程对 volatile 进行读取时,要从主内存读取最新的数据

防止指令重排序:

volatile 关键字通过“内存屏障”来防止指令被重排序.

为了实现 volatile 的内存语义,编译器在生成字节码时,会在指令序列中插入内存屏障来禁止特定类型的处理器重排序.然而对于编译器来说,发现一个最优布置来最小化插入屏障的总数几乎不可能,为此,java 内存模型采取保守策略

下面是基于保守策略的 JMM 内存屏障插入策略:

  • 在每个 volatile 写操作的前面插入一个 StoreStore 的屏障
  • 在每个 volatile 写操作的后面插入一个 StoreLoad 屏障
  • 在每个 volatile 读操作的前面插入一个 LoadLoad 屏障
  • 在每个 volatile 读操作的后面插入一个 LoadStore 屏障
